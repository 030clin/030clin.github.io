## 📌 프로젝트 소개

![Nae-il 메인 화면](/assets/img/naeil-main.png)

Nae-il은 프로그래밍 학습을 위한 LMS(Learning Management System) 플랫폼으로,  
강의 수강, 실시간 강의, 구독 기반 학습을 하나의 서비스로 통합한 프로젝트이다.

기존 LMS가 가지고 있던 단순 강의 제공 구조를 넘어,  
학습 지속성과 사용자 참여를 높이는 것을 목표로 설계되었다.

- 학생: 강의 수강 및 학습 관리  
- 강사: 강의 개설 및 운영  
- 관리자: 서비스 운영 및 품질 관리  

---

## 🚨 문제 정의

프로젝트를 시작하며 기존 LMS 구조에서 다음과 같은 문제를 발견했다.

### 1. 학습 참여도 저하
기존 온라인 강의는 녹화 영상 중심으로 제공되어  
학습자 간 상호작용이 부족하고 몰입도가 떨어지는 문제가 있었다.

### 2. 학습 지속성 부족
단건 결제 구조로 인해 학습 흐름이 단절되고  
사용자의 재방문을 유도하기 어려웠다.

### 3. 실시간 강의 운영 비효율
예약/취소 기준이 없어 무분별한 신청과 노쇼 문제가 발생했다.

### 4. 강의 품질 관리 부재
강의 개설에 대한 검증 구조가 없어  
품질이 낮은 콘텐츠도 등록되는 문제가 있었다.

![기존 LMS 문제 구조](/assets/img/lms-problem.png)

👉 결국 이 프로젝트는  
**“학습 지속성과 서비스 신뢰성을 개선하는 것”을 목표로 시작되었다.**

---

## ⚙️ 해결 전략

위 문제를 해결하기 위해 다음과 같은 구조를 설계했다.

- 실시간 강의 시스템 도입 → 사용자 간 상호작용 강화  
- 구독 기반 모델 도입 → 지속적인 학습 유도  
- 예약 / 취소 정책 설계 → 운영 효율 개선  
- 관리자 승인 구조 도입 → 강의 품질 보장  

👉 단순 기능 추가가 아니라  
**“문제 해결 중심 설계”**를 목표로 접근했다.

---

## 🏗 시스템 구조

![ERD 구조](/assets/img/system-architecture.png)

서비스 전반의 데이터 흐름을 고려하여 ERD를 설계하였다.  
각 도메인 간 관계를 명확히 정의함으로써 데이터 정합성과 확장성을 확보하였다.

- Spring Boot 기반 서버 구조  
- Spring Security 기반 권한 관리  
- JPA를 통한 데이터 접근  
- MySQL 기반 관계형 데이터 설계  

👉 이를 통해  
**확장 가능하고 유지보수 가능한 구조를 구축하였다.**

## 🧠 기술적 문제 해결

### 🚨 문제 상황: 장바구니 중복 및 재구매 허용

![문제 상황](/assets/img/cart-problem.png)

장바구니 기능이 단순 INSERT 구조로 설계되어 있어  
동일 강의의 중복 담기 및 이미 구매한 강의의 재구매가 가능했다.

이로 인해 데이터 정합성이 깨지고 사용자 혼란이 발생할 수 있었다.

---

### ⚙️ 해결 방식: 데이터 정합성 보장 구조 설계

![해결 구조](/assets/img/cart-solution.png)

Service 레벨 검증과 DB 레벨 제약을 동시에 적용하여  
동시성 상황에서도 데이터 무결성을 보장하도록 설계하였다.

- Service 레벨: 비즈니스 로직 기반 중복 검증  
- DB 레벨: Unique 제약 조건 설정  

---

### 📈 결과

- 동일 강의 중복 담기 방지
