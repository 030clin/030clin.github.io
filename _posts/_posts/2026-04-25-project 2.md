## 📌 프로젝트 소개

Nae-il은 프로그래밍 학습을 위한 LMS(Learning Management System) 플랫폼으로,  
강의 수강, 실시간 강의, 구독 기반 학습을 하나의 서비스로 통합한 프로젝트이다.

기존 LMS가 가지고 있던 단순 강의 제공 구조를 넘어,  
학습 지속성과 사용자 참여를 높이는 것을 목표로 설계되었다.

- 학생: 강의 수강 및 학습 관리  
- 강사: 강의 개설 및 운영  
- 관리자: 서비스 운영 및 품질 관리  

---

## 🚨 문제 정의

프로젝트를 시작하며 기존 LMS 구조에서 다음과 같은 문제를 발견했다.

### 1. 학습 참여도 저하
기존 온라인 강의는 녹화 영상 중심으로 제공되어  
학습자 간 상호작용이 부족하고 몰입도가 떨어지는 문제가 있었다.

### 2. 학습 지속성 부족
단건 결제 구조로 인해 학습 흐름이 단절되고  
사용자의 재방문을 유도하기 어려웠다.

### 3. 실시간 강의 운영 비효율
예약/취소 기준이 없어 무분별한 신청과 노쇼 문제가 발생했다.

### 4. 강의 품질 관리 부재
강의 개설에 대한 검증 구조가 없어  
품질이 낮은 콘텐츠도 등록되는 문제가 있었다.

👉 결국 이 프로젝트는  
**“학습 지속성과 서비스 신뢰성을 개선하는 것”을 목표로 시작되었다.**

---

## ⚙️ 해결 전략

위 문제를 해결하기 위해 다음과 같은 구조를 설계했다.

- 실시간 강의 시스템 도입 → 사용자 간 상호작용 강화  
- 구독 기반 모델 도입 → 지속적인 학습 유도  
- 예약 / 취소 정책 설계 → 운영 효율 개선  
- 관리자 승인 구조 도입 → 강의 품질 보장  

👉 단순 기능 추가가 아니라  
**“문제 해결 중심 설계”**를 목표로 접근했다.

---

## 🏗 시스템 구조

서비스는 다음과 같은 구조로 설계되었다.

- Spring Boot 기반 서버 구조
- Spring Security를 통한 RBAC 권한 관리
- JPA를 활용한 데이터 관리
- MySQL 기반 관계형 데이터 설계

또한 Controller - Service - Repository 계층 구조를 통해  
각 레이어의 책임을 분리하였다.

👉 이를 통해  
**확장 가능하고 유지보수 가능한 구조를 확보할 수 있었다.**

## 🔥 핵심 기능

### ✔ 실시간 강의 예약 시스템
동시성 문제가 발생할 수 있는 예약 기능에서  
트랜잭션과 락을 활용하여 정원 초과를 방지하였다.

---

### ✔ 구독 기반 학습 모델
사용자가 일정 기간 동안 강의를 자유롭게 수강할 수 있도록 하여  
학습 지속성을 확보하였다.

---

### ✔ 블랙리스트 정책
무분별한 예약 취소를 방지하기 위해  
경고 누적 시스템과 블랙리스트 기능을 구현하였다.

---

### ✔ 관리자 승인 시스템
강의 등록 후 바로 활성화되지 않고  
관리자의 승인을 거쳐 서비스에 반영되도록 설계하였다.

👉 이를 통해  
**서비스 신뢰도를 높이는 구조를 만들었다.**

## 🧠 기술적 문제 해결

### 1. API 구조 유지보수 문제

#### 문제
기능이 증가하면서 API 구조가 복잡해지고  
유지보수가 어려워지는 문제가 발생했다.

#### 해결
- 도메인 기반 패키지 구조 설계  
- Controller / Service / Repository 계층 분리  
- DTO 기반 요청/응답 구조 정리  

#### 결과
- 기능 추가 시 영향 범위 감소  
- 협업 효율 증가  
- 유지보수성 향상  

---

### 2. 예외 처리 중복 문제

#### 문제
각 Controller마다 예외 처리 방식이 달라  
코드 중복이 발생했다.

#### 해결
- GlobalExceptionHandler 도입  
- @ControllerAdvice 기반 전역 예외 처리  

#### 결과
- 코드 중복 제거  
- 에러 응답 일관성 확보  
- 디버깅 효율 향상  

---

### 3. 공통 로직 중복 문제

#### 문제
로그 처리 및 성능 측정 코드가  
여러 서비스에 반복적으로 작성되었다.

#### 해결
- AOP 적용  
- 공통 관심사 분리  

#### 결과
- 코드 가독성 향상  
- 유지보수성 개선  

---

## 💡 개인 기여 (핵심 경험)

### 🚨 문제 상황
장바구니 기능에서 다음과 같은 문제가 발생했다.

- 동일 강의 중복 담기 가능  
- 이미 구매한 강의 재구매 가능  
- 결제 시 중복 결제 발생 가능  

👉 이는 데이터 정합성을 깨뜨리고  
사용자 경험을 저하시킬 수 있는 문제였다.

---

### ⚙️ 해결 방식

데이터 정합성을 보장하기 위해  
다음과 같은 구조를 설계했다.

#### 1. Service 레벨 검증
- 비즈니스 로직 기반 중복 체크

#### 2. DB 레벨 제약
- Unique 제약 조건 추가

👉 이중 검증 구조를 통해  
동시성 상황에서도 안정성을 확보했다.

---

### 📈 결과

- 중복 결제 방지  
- 데이터 무결성 확보  
- 사용자 혼란 방지  

👉 이 경험을 통해  
**데이터 정합성이 백엔드에서 얼마나 중요한지 이해하게 되었다.**
