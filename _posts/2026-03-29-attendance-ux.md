## 🚨 문제 정의

LMS 출결 관리 시스템을 개발하는 과정에서  
출결 상태를 DB에는 영어 코드로 저장하고,  
이를 그대로 사용자 화면에 출력하는 구조를 사용하였다.

출결 상태는 다음과 같이 구성되어 있었다.

- 출석
- 지각
- 결석
- 공결

하지만 DB에서는 이를 영어 값으로 관리하고 있었고,  
해당 값이 그대로 화면에 출력되면서  
사용자 입장에서 직관성이 떨어지는 문제가 발생하였다.

즉,  
**시스템 중심의 데이터 구조가 사용자 경험을 저해하는 상황이었다.**

---

## 🧱 기존 구조 (Before)


출결 상태가 DB에서 영어 값으로 저장되고  
별도의 처리 없이 그대로 사용자 화면에 출력되는 구조였다.

---

## 🎯 목표

- DB 구조를 변경하지 않고 유지
- 사용자에게 직관적인 한글 상태 제공
- 전체 조회, 주차별 조회, 유형별 조회 등 모든 기능에서 일관된 UX 유지

---

## 🤔 해결 전략

출결 상태 처리 방식에 대해 다음 두 가지를 고려하였다.

### 1. DB를 한글로 변경
- 구현은 단순
- 하지만 데이터 일관성 및 확장성 저하 가능

### 2. 로직에서 변환 처리 (선택)

👉 DB는 영어 코드 유지  
👉 출력 단계에서 한글로 변환

---

## ⚙️ 해결 방법

출결 상태를 출력하는 모든 지점에서  
영어 상태값을 한글로 변환하는 로직을 추가하였다.

```java
public String convertToKoreanStatus(String status) {
    switch (status) {
        case "PRESENT": return "출석";
        case "LATE": return "지각";
        case "ABSENT": return "결석";
        case "OFFICIAL": return "공결";
        default: return "알 수 없음";
    }
}
